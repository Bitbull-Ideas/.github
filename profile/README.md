# Hermes Agent & Chris Ruettimann Workspace

This is a workspace for collaboration between **Chris Ruettimann** and **Hermes Agent**.

It is a place for ideas, experiments, notes, runbooks, prompts, and practical workflows around human-agent collaboration. The goal is simple: make both Chris and Hermes better at doing useful engineering work together.

His work focuses on:

- Linux systems and service operations
- Ansible automation
- Infrastructure troubleshooting
- Application debugging
- Logging and observability
- Repeatable operational workflows
- Turning manual fixes into maintainable automation

Chris values practical results over theory. A solution should be understandable, testable, repeatable, and useful on real systems.

## About Hermes Agent

Hermes Agent is Chris's AI engineering partner.

Hermes should help by:

- Asking better questions
- Checking assumptions
- Producing concrete artifacts
- Verifying work with real output
- Turning repeated lessons into reusable procedures
- Helping Chris think more clearly about complex tasks

## Purpose of This Workspace

This workspace exists to improve how Chris and Hermes collaborate.

It can be used to collect:

- README drafts
- Workflow ideas
- Agent instructions
- Prompt patterns
- Verification checklists
- Operational runbooks
- Automation notes
- Lessons learned from real infrastructure work

The idea is to build a feedback loop:

1. Chris brings real-world context, judgment, and experience.
2. Hermes helps investigate, structure, write, automate, and verify.
3. Useful results are captured as reusable knowledge.
4. Future work starts smarter than the last session.

## Collaboration Principles

### Verify Before Claiming

Do not assume something works. Check it.

A good answer should be backed by evidence: command output, logs, tests, rendered files, Git state, service status, or another concrete verification step.

### Prefer Useful Artifacts

A conversation is good. A reusable artifact is better.

The best sessions should leave behind something practical: a script, checklist, runbook, README, role improvement, test result, issue summary, or documented decision.

### Keep Things Maintainable

Automation and documentation should be clear enough that another person can understand them later.

Simple, readable, and reliable is better than clever and fragile.

### Coach Each Other

This workspace is not only for producing output. It is also for improving the collaboration itself.

Chris can use it to improve how he delegates, explains context, and reviews results. Hermes can use it to improve how it investigates, verifies, documents, and learns from repeated work.

### Human Judgment Comes First

Hermes can accelerate work, but Chris remains responsible for intent, context, risk, and final decisions.

The agent should support good engineering judgment, not hide uncertainty behind confident language.

## What Belongs Here

Good content for this workspace includes:

- Human-agent collaboration notes
- Prompt and instruction templates
- Troubleshooting patterns
- Engineering checklists
- Runbook drafts
- Small helper scripts
- Automation ideas
- Lessons from real incidents or projects

Do not store:

- Secrets
- Passwords
- API tokens
- Customer data
- Private infrastructure details that do not belong in a public repo
- Temporary logs without long-term value
- Unverified claims presented as facts
